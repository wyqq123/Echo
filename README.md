# 产品定位
一款面向学生党和工作族的日程管理效率产品，核心解决信息爆炸背景下的用户”决策焦虑“和被各种社交媒体产品分割注意力环境下的“时间失忆”痛点
1. **决策层**：帮用户从“繁杂待办”中筛选出“核心三件事”。
2. **执行层**：帮用户感知并利用好“通勤碎片时间”。

# 功能模块

## Focus Funnel：认知漏斗
### 核心解决痛点
- **认知卸载：** 让用户把脑子里想到要做的事情，轻松、不需要整理和思考地说出来
- **解决“决策瘫痪”：** 针对由于选项过多导致的大脑前额叶停摆，通过漏斗逻辑强行收窄选择域。
- **消除“未完成焦虑”：** 允许用户“乱写一通”，通过AI的分类整理和工作流拆解给用户一种“事情已被妥善安置”的掌控感。
- **对抗“伪勤奋”：** 识别低价值的碎片任务，强制用户直面长期目标，敢于对无效忙碌做减法
### 主要功能
- **AI任务拆解：** Echo会自动拆解你输入的所有事项，并进行主题类型映射、完成时间预估以及工作流拆解，其中工作流的拆解主要有线性和维度两种拆解方式
- **决策四问：** 当Echo检测到你今天输入的事项>=4时，会触发决策四问机制，通过“减法问-杠杆问-破冰问-边界确认问”，帮助你筛选出今天最需要关注/与季度设定目标最相关的三件锚点事项，守护你的注意资源
- **事项卡片合并与拆分：** AI会给你提供处理好的待办事项卡片，你可以在此基础上轻松做任务的合并、拆解和编辑
- **事项卡片自动创建：** 当确认无误后，事项卡片会自动创建在Fluid Timeline的时间轴/任务抽屉中

## Fluid Timeline
### 核心解决痛点
- **计划的脆弱性（Fragility）：** 传统日历一旦某个环节被打断（如突发会议），后续所有计划会因“失效红点”导致用户产生“全或无（All-or-Nothing）”的弃疗心理。
- **执行压力：** 密密麻麻的固定时间表会产生视觉压迫。通过“高光锚点”与“任务池”的结合，释放用户的即时认知压力。
### 主要功能
- **创建与编辑任务：** 你可以直接在时间轴上右击空白区块来创建新任务；你可以通过右击已有的事件区块来编辑已经创建的任务，也可以自主决定是否要把它作为你的锚点事项
- **拖拽任务 + 任务抽屉：** 你可以直接在时间轴上拖拽任务来调整事件块的事件相对位置，下方事件块会自动顺延~ 任务抽屉主要放置的是那些决策四问中所确认的非锚点事项，当然，你完全能把任务抽屉中的待办事项直接拖拽到时间轴上！
- **决策冰柜：** 这里会存储你过去没有勾选完成的锚点任务，你可以手动把任务“复活”到任务抽屉中，也可以直接清除这些任务；同时，它们也会作为决策四问的来源事项，让你在决策中回顾过去可能很重要的未完成事项~

## Compute Mod
### 核心解决痛点
- **时间失忆 (Time Amnesia)：** 解决“刷了半小时短视频，放下手机却想不起做了什么”的空虚感。
- **决策疲劳 (Decision Fatigue)：** 在通勤这种低意志力时刻，提供有限的选择，防止用户因不知道干什么而下意识滑向成瘾性应用。
- **心理负担补偿：** 即使是玩游戏或刷剧，只要是“有意识的选择”，就能消除你对“荒废时间”的负罪感，将其转化为“补能体验”。
### 主要功能
- **三种能量状态的沉浸模式：**包括生产（处理碎片化工作任务与信息）、增长（看书、听博客等丰富个人知识的活动）、恢复（刷视频、玩游戏等休闲娱乐活动）。
- **沉浸倒计时：**选择一个最契合你当下的能量状态，进入沉浸仓界面，默认15min倒计时，当倒计时耗尽会弹窗 + 提示音提示。
- **倒计时溢出处理：**当15分钟的任务时间耗尽后，不直接停止，而是进入“溢出模式”（沉浸仓界面变色），溢出达到 15 分钟用户仍未操作，系统自动发送通知并退出沉浸仓界面。当你再次刷新该界面时，系统会弹窗提示“需要录入溢出时间 or 舍弃溢出时间”。


## Echo Compass
### 核心解决痛点
- **看到价值沉淀：** 从执行闭环到认知闭环：看见你的时间质量和长期价值沉淀：你完成了哪些锚点事项，它们如何帮助你的目标达成；你的碎片化时间和注意力都被用在哪里了。
### 主要功能
- **锚点任务完成情况：**查看你当天最重要的3件事的完成情况。
- **通勤胶囊沉浸时间分布：**查看你该季度的通勤或是其他碎片化场景下，生产/恢复/增长这三种能量状态的用时分布情况。
- **任务森林：**查看你该季度下主要聚焦的三大目标树下的锚点事项完成情况，并根据Echo AI助手提供的季度报告来做计划和精力分配的进一步迭代与优化！
